# PAGE FAMILY UX RULES

## 1. Homepage (`app/(landing)`)
- **Hero:** Intent-first. Strong primary pathways. Never vague brand fluff.
- **Proof:** Show the `ProofStrip` and `CommandMap` coverage stats above the fold.
- **Flow:** Trust → Discovery → Action.

## 2. Directory Pages (`app/directory`)
- **Scan Speed:** TanStack tables and scan-optimized row cards. High density.
- **Filters:** Sticky layout. On mobile, use native-feeling bottom sheets instead of clunky inline accordions. 
- **Validation:** Trust ratings, live availability, and statuses must be highly visible without creating visual noise.

## 3. Profiles / Detail Pages (`app/[entities]`)
- **Hero Shell:** Provide immediate identity, geography, coverage area, and the primary "Claim" or "Connect" CTA.
- **Trust Elements:** Verification status must be unmissable.

## 4. Content / SEO (Tools, Regs, Blog)
- **Reading:** Ideal line widths (65-75 chars).
- **Structure:** Best-in-class typography. Sidebars and anchors that look like a SaaS doc site, not a generic WordPress theme.
- **No Dead Ends:** `NoDeadEndBlock` must be implemented at the bottom of every tool and regulation page.

## 5. Map & Leaderboards
- **Visuals:** Performance-minded. Ranking visuals should feel game-like but retain hardcore commercial credibility.
